# 智源Talk 内容生成器

根据用户提供的素材，生成智源Talk活动介绍文案，并在审核通过后生成标准格式的 md 文件。

## 功能概览

| 阶段 | 功能 | 说明 |
|-----|------|------|
| **阶段一** | 生成内容介绍文案 | 将研究简介转化为标准格式的宣传文案 |
| **阶段 2.5** | 生成吸睛标题（可选） | 内容介绍通过后，询问用户是否需要生成营销风格标题 |
| **阶段二** | 生成 md 文件 | 审核通过后，生成完整的智源活动 md 文件 |

## 阶段一：生成内容介绍文案

### 输入素材清单

用户需要提供以下信息：

| 字段 | 来源 | 示例 |
|-----|------|------|
| **期数** | 用户提供 | 355期 |
| **时间** | 用户提供 | 4月22日 上午10:30-11:30 |
| **主讲人** | 从嘉宾简介提取 | 周莉，香港中文大学（深圳） |
| **研究简介** | 用户提供 | 论文标题、摘要、主要贡献 |
| **发表会议** | 用户提供或推断 | ICLR 2026 |
| **演讲形式** | 用户提供或默认 | 线上分享 |


### 输出模板（内容介绍文案）

**⚠️ 硬性约束：内容介绍文案总字数不得超过 230 字（含标点）。**

生成文案遵循以下结构，每部分需高度凝练：

```
【背景趋势/痛点问题】（1句，精准点题，不铺开，约40-50字）

【智源Talk信息 + 研究内容】智源Talk XXX期邀请了XXX【机构】【线上/线下】分享，工作来自XX会议/期刊收录论文《XXX》。（紧凑嵌入，不单独占句。**如果用户未提供会议/期刊信息，则省略"工作来自…"这句，不写"预印本"等推测性表述**）

【研究亮点】（1-2句，只概括核心贡献和意义，不罗列具体任务细节/评测数字/方法步骤，约80-100字，用"提出了..."、"搭建了..."、"为...提供了..."等句式）

【收尾】欢迎大家一起讨论交流。
```

### 写作要点

- **背景**：1句搞定，直击痛点，不要"取得了快速进展，但..."这种铺垫
- **研究亮点**：只说做了什么、为什么重要，不说具体怎么做的（任务列表、评测数量、实验细节等一律省略）
- **节奏**：背景短→信息紧凑→亮点精炼→收尾轻快，总字数 ≤ 230 字
- **学术感**：避免营销味，用"提出了...为...提供了..."等学术句式

### 内容介绍文案示例

**示例 1（FeatureBench）**：

> 大模型赋能下，Agentic Coding 成为新一代代码开发范式，而现有评测基准的任务设计、场景适配性已难以匹配真实开发需求。智源Talk 355期邀请了西安交通大学张家铖线上分享，工作来自ICLR 2026收录学术论文《FeatureBench: Benchmarking Agentic Coding for Complex Feature Development》。提出了全新的 FeatureBench 评测框架，立足实际开发场景打造高质量评测基准，同时搭建起全流程开源的技术基础设施，既为当下 Code Agent 的能力评估提供了科学标准，也为后续相关模型的训练与优化筑牢了数据基础。欢迎大家一起讨论交流。（217字）

**示例 2（EchoMind）**：

> 语音大模型在口语理解上进展迅速，但能否同时理解语言内容与语音中的情绪线索并做出共情回应，仍是开放问题。智源Talk 357期邀请了香港中文大学（深圳）周莉线上分享，工作来自ICLR 2026收录论文《EchoMind: An Interrelated Multi-Level Benchmark for Evaluating Empathetic Speech Language Models》。提出了首个用于评估共情型语音语言模型的多层级关联基准，通过相互关联的任务模拟共情对话的认知过程，为系统评估语音模型的情感智能提供了新范式，揭示了当前模型在高表现力语音理解方面的关键挑战。欢迎大家一起讨论交流。（208字）

## 阶段 2.5：生成吸睛标题（可选）

内容介绍文案审核通过后，**主动询问用户是否需要生成吸睛标题**。

### 生成规则

**⚠️ 硬性约束：吸睛标题总字数不得超过 45 字（含标点）。**

根据素材信息，从以下策略中组合使用（可叠加，但总数不超过 45 字）：

| 策略 | 适用场景 | 示例 |
|-----|---------|------|
| **顶会标签前置** | 来自顶会（ICLR/CVPR/NeurIPS/ACL 等） | `ICLR 2026｜` |
| **机构/公司噱头前置** | 来自知名公司（Google、Meta 等）或热门高校 | `Google出品｜` `北大团队｜` |
| **学术标题"营销化"** | 把学术标题改写为大众易懂、引发好奇的表达 | `让AI学会写论文？` `你的代码Agent真的能干活吗？` |

### 写作要点

- 语言简单直白，面向大众而非学术圈
- 不犯学术错误，不夸大、不误导
- 吸引眼球但保持底线，不要标题党到虚假的程度
- 可以用问句、感叹号、对比句式增加吸引力

### 吸睛标题示例

| 原学术标题 | 吸睛标题 |
|-----------|---------|
| EchoMind: An Interrelated Multi-Level Benchmark for Evaluating Empathetic Speech Language Models | ICLR 2026｜你的AI懂你的情绪吗？首个语音共情评测基准来了 |
| FeatureBench: Benchmarking Agentic Coding for Complex Feature Development | ICLR 2026｜AI写代码能力到底行不行？交大团队给出答案 |
| PaperOrchestra: A Multi-Agent Framework for Automated AI Research Paper Writing | Google出品｜AI帮你写论文？多智能体框架实现自动成稿 |


## 注意事项

- **⚠️ 内容介绍文案总字数不得超过 230 字（含标点），这是硬性约束**
- 背景只用1句精准点题，不铺垫不展开
- 研究亮点只概括核心贡献和意义，不罗列具体任务细节、评测数字、实验步骤
- 内容介绍文案应简洁专业，避免过度营销感
- 研究亮点使用"提出了..."、"搭建了..."、"为...提供了..."等学术表达
- TALK标题需符合智源公众号的学术调性，包含论文级别标签（如 ICLR、CVPR 等）
- 公众号标题为营销风格吸睛标题（≤45字），与 TALK 标题分开，由阶段 2.5 生成
- md 文件 frontmatter 中的 date 格式为 ISO 8601（YYYY-MM-DD HH:MM:SS）
- 固定信息（主持人、报告地点）直接使用模板值，无需用户确认
- 如素材信息不足，先询问用户补充关键信息再生成
- md 文件统一输出到 `~/Desktop/智源talk/` 文件夹，每期独立一个 md 文件
- 文件命名格式：`智源Talk_XXX期_关键词.md`（如 `智源Talk_357期_EchoMind.md`）
- 如 `~/Desktop/智源talk/` 文件夹不存在，先自动创建
